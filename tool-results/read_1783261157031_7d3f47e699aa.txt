    28→# PART 1 — PRODUCT REQUIREMENTS (PRD)
    29→
    30→## 1.1 Vision
    31→Kedbyte Payroll is a multi-tenant UK payroll bureau platform: one bureau operates payroll for many client companies, each with many employees, with HMRC RTI compliance built in as a first-class workflow — not an export. Two surfaces: the **Bureau Command Center** (dark "Void" professional console) and the **Employee Self-Service Portal** ("My Pay"). Design promise: *the system is not perceived as fast; it is perceived as present.*
    32→
    33→## 1.2 Users & Roles
    34→
    35→| Role | Surface | Core jobs |
    36→|------|---------|-----------|
    37→| `bureau_admin` | Bureau | Everything across all client companies: onboarding, pay runs, RTI, pensions, settings |
    38→| `company_admin` | Bureau (scoped) | Own company: employees, pay runs, approvals |
    39→| `payroll_manager` | Bureau (scoped) | Assigned companies: run + review payroll, no settings |
    40→| `employee` | ESS Portal | Payslips, P60s, holidays, personal details |
    41→| `manager` (employee flag) | ESS Portal | + approve direct reports' leave |
    42→| `accountant` | Bureau (read) | Financial reports, aggregates only |
    43→
    44→## 1.3 Feature Scope (v1)
    45→1. Multi-company management with HMRC reference validation & bank modulus checking
    46→2. Employee lifecycle: starter (A/B/C declarations, P45 intake) → active → leaver (P45 out via FPS)
    47→3. 4-step Pay Run Wizard with server-side calculation engine (PAYE/NI/SL/pension/statutory), variance detection, approval, FPS generation
    48→4. RTI: FPS + EPS submission via GovTalk Transaction Engine, poll lifecycle, error resolution engine, resubmission
    49→5. Pension auto-enrolment: per-period assessment, opt-out + refund, NEST/PAPDIS contribution files
    50→6. Payslip/P60/P45 generation (print-grade), document vault with hash verification
    51→7. ESS: payslips, holiday requests + manager approval chain, personal details (24h bank-change cooling-off), documents, notifications
    52→8. Reports: gross-to-net, P32-style employer summary, gender pay gap, pension contributions, variance
    53→9. Pay schedule + bank holiday + BACS calendar engine (gov.uk sync)
    54→10. Settings with **effective-dated statutory rate tables** (historical recalculation always deterministic)
    55→11. Hash-chained immutable audit ledger on every mutation
    56→12. Auth: Argon2id + TOTP MFA + rotating refresh tokens + account recovery
    57→
    58→Out of scope v1 (phase 2): CIS, attachment-of-earnings orders, apprenticeship levy reporting, multi-currency, Confirmation of Payee, payrolled benefits (P11D flows), Scottish/NI bank-holiday divisions beyond data model.

   150→# PART 4 — DATABASE SCHEMA (PostgreSQL, production DDL)
   151→
   152→Multi-tenancy: shared schema + `tenant_id` on every row + **RLS as backstop** + app-layer scoping. Under PgBouncer transaction pooling, tenant context MUST be set per-transaction: `SELECT set_config('app.current_tenant', $1, true)` (SET LOCAL semantics) at the start of every transaction — session-level SET leaks across pooled connections.
   153→

   169→  password_hash TEXT NOT NULL,              -- Argon2id m=64MB t=3 p=4

   171→    ('bureau_admin','company_admin','payroll_manager','employee','accountant')),

   455→## 5.2 Money rounding rules (HMRC conventions — encode exactly)
   456→
   457→```
   458→floorPound(v)  = ⌊v⌋            → taxable pay to date; student loan deduction
   459→floorPenny(v)  = ⌊v·100⌋/100    → tax due to date (truncate)
   460→niRound(v)     = nearest penny, exact half-penny rounds DOWN   → every NI product
   461→round2(v)      = commercial 2dp → pension contributions, display sums
   462→```

   464→## 5.3 Tax code parser — `parseTaxCode(input: string): ParsedTaxCode`
   465→
   466→```
   467→Normalise: uppercase, collapse spaces.
   468→W1/M1 detection: trailing " W1" | " M1" | "X"  → week1Month1=true, strip.
   469→Regime: leading 'S' → scotland; leading 'C' → wales (rates mirror rUK today; keep flag).
   470→Flat codes: BR 20% · D0 40% · D1 45% · SBR 20% · SD0 21% · SD1 42% · SD2 45% · SD3 48% · CBR/CD0/CD1.
   471→NT → no tax.   0T → zero free pay, full progressive bands.
   472→K-codes: K{n} → annualAdditionalPay = n×10+9  (ADDED to taxable).
   473→Suffix codes: {n}[L|M|N|T] → annualFreePay = n×10+9   ← Table A convention: 1257L = £12,579, NOT £12,570.
   474→Unknown → throw (surface as field validation).
   475→```

   477→## 5.4 PAYE — `calculatePAYE({taxCode, period, grossTaxableThisPeriod, ytdTaxable, ytdTaxPaid}) → {tax, taxablePayToDate, freePayToDate, regulatoryLimited, code}`
   478→
   479→Cumulative method (default):
   480→```
   481→p            = tax month 1..12         (W1/M1 ⇒ p=1, YTD figures ignored)
   482→grossToDate  = ytdTaxable + grossThis
   483→freePayToDate= annualFreePay × p / 12                 (round2)
   484→addlToDate   = annualAdditionalPay × p / 12           (K codes)
   485→taxableToDate= floorPound(max(0, grossToDate − freePayToDate + addlToDate))
   486→bands pro-rated: bandCeiling_toDate = annualCeiling × p/12
   487→taxDueToDate = Σ slice×rate over pro-rated bands      (floorPenny)
   488→tax          = taxDueToDate − ytdTaxPaid              (negative ⇒ REFUND through payroll)
   489→REGULATORY LIMIT: tax ≤ 50% × grossTaxableThisPeriod  (K & 0T protection; excess carries fwd)
   490→```
   491→Flat codes: `taxDueToDate = floorPenny(floorPound(grossToDate) × flatRate)`.
   492→Taxable gross definition: gross earnings − net-pay-arrangement pension − salary-sacrifice; **relief-at-source pension does NOT reduce taxable pay**.
   493→
   494→New-starter code assignment (no P45): Statement A → 1257L cumul · B → 1257L W1/M1 · C → BR · none → 0T W1/M1. With P45: use P45 code + prior pay/tax as opening YTD (unless code is BR/0T/D-prefix → don't carry pay/tax).
   495→
   496→## 5.5 National Insurance — `calculateNI(E, category) → {employee, employer, earningsAtLEL, earningsLELtoPT, earningsPTtoUEL}`
   497→
   498→Per-period, non-cumulative (except directors). NI-able gross = gross − salary-sacrifice (RAS and net-pay pension do NOT reduce NI-able).
   499→```
   500→EMPLOYEE:                                    EMPLOYER (15% above threshold):
   501→A,H,M,V: 8%×(min(E,UEL)−PT)₊ + 2%×(E−UEL)₊   A,B,C,J: threshold = ST (£417)
   502→B      : 1.85% main band + 2% upper           H,M,V,Z: threshold = UEL £4,189 (0% below — replaces ST)
   503→J,Z    : 2% × (E−PT)₊  (deferment flat)       Freeport letters: threshold = £2,083
   504→C,X    : 0                                    X: 0
   505→Every product niRound()ed (nearest penny, half DOWN).
   506→```
   507→FPS band reporting (mandatory even when NI=0): earnings at LEL, LEL→PT, PT→UEL — sub-PT employees still earn State Pension qualifying years and MUST appear on FPS.
   508→
   509→Directors — `calculateDirectorNI(cumulativeEarnings, category, eePaidYTD, erPaidYTD, weeksAsDirector)`:
   510→annual thresholds (PT £12,570 / UEL £50,270 / ST £5,000) pro-rated by weeks/52 for mid-year appointments; compute NI on cumulative earnings, subtract YTD paid. Alternative (table) method = normal monthly + mandatory annual true-up in final period.
   511→
   512→Employment Allowance: cumulative employer-NI offset up to £10,500/scheme/year, consumed run-by-run, claimed via EPS flag; never offsets Class 1A/1B.

   514→## 5.6 Student loans — `calculateStudentLoan(E, plan)` → `floorPound(max(0, E − threshold_m) × rate)`
   515→
   516→Monthly thresholds: P1 £2,241.67 · P2 £2,448.75 · P4 £2,816.25 · P5 £2,083.33 · PGL £1,750.00. One undergraduate plan max; **postgraduate runs CONCURRENTLY** (separate 6% deduction). Started by DPS SL1/PGL1, stopped by SL2/PGL2. Base = NI-able gross.

   518→## 5.7 Pensions — `calculatePension(E, scheme) → {pensionableEarnings, eeGross, eeDeducted, er, reducesTaxable, reducesNIable}`
   519→
   520→```
   521→basis = qualifying_earnings: base = max(0, min(E, 4189.17) − 520)
   522→        pensionable_full | total_earnings: base = E
   523→eeGross = base × eeRate      er = base × erRate
   524→relief branching:
   525→  relief_at_source : eeDeducted = eeGross × 0.80  (provider reclaims 20%); taxable & NI-able unchanged
   526→  net_pay          : eeDeducted = eeGross; reducesTaxable = eeGross      (NI-able unchanged)
   527→  salary_sacrifice : eeDeducted = 0; er += eeGross; reduces BOTH taxable and NI-able
   528→```
   529→Assessment — `assessAutoEnrolment(age, monthlyEarnings, spa=67)`:
   530→eligible (age 22–SPA AND earnings>£833/mo → AUTO-ENROL) · non_eligible (16–74 with earnings>trigger outside age band, or earnings in QE band → may opt in, employer must contribute) · entitled (16–74, ≤QEL → may join, no employer duty). SPA rises 66→67 between Apr 2026–Mar 2028 by DOB — implement `getStatePensionAge(dob)` from GOV.UK timetable, not a constant. Postponement ≤3 months. Opt-out window 1 month → full refund via negative deduction; re-enrolment every 3 years.

   940→## 8.3 IRmark — `computeIRmark(bodyXml: string): { base64: string; base32: string }`
   941→```
   942→1. Extract the <Body> element; the GovTalk envelope namespace must be INHERITED onto it
   943→   (missing/stray namespaces on Body = the classic IRmark-invalid rejection).
   944→2. Blank the IRmark element's text content (element remains).
   945→3. Canonicalise with standard W3C XML C14N 1.0 (without comments).
   946→4. digest = SHA-1(canonical bytes) → 20 raw bytes.
   947→5. Embedded IRmark = base64(digest) [28 chars]; user-visible receipt form = base32(digest).
   948→HMRC's own step-by-step guide: do NOT hand-roll C14N — use a proven library (xml-crypto, libxml).
   949→Demo build: we emit deterministic canonical-form XML so base64(sha1(body)) is exact;
   950→production swaps in library C14N behind the same function signature.
   951→```

   953→## 8.4 Submission worker state machine (queues `rti:submit`, `rti:poll`)
   954→```
   955→submit : build XML → IRmark → POST /submission
   956→   ← acknowledgement {CorrelationID, ResponseEndPoint{PollInterval}}
   957→   persist correlation_id · status=polling · poll_after = now + PollInterval
   958→poll   : POST /poll {Class, Qualifier:"poll", Function:"submit", CorrelationID}
   959→   ← acknowledgement (still processing) ⇒ backoff ×1.5, cap 30 min, attempts ≤ 20
   960→   ← response ⇒ business ACCEPTED → send delete-request → status=accepted → WS + notify
   961→   ← error    ⇒ parse GovTalkErrors[] → rti_errors rows → status=rejected → S21 workflow
   962→transport failure ⇒ retry 1m/5m/30m/2h/8h → dead-letter + ops alert
   963→HMRC auto-deletes correlations after 60 days — treat missing correlation on poll as expired.
   964→```
